OpenFlights FAQ

General

What is OpenFlights?

In a nutshell, it's a tool that lets you map your flights around the world, search and filter them in all sorts of interesting ways, and calculate statistics automatically.

Why is OpenFlights better than other similar services?

Lots of reasons!
  • OpenFlights has a way cool map. You can pan, zoom, select, scroll and explore all you like!
  • OpenFlights is user-friendly and efficient. Everything's on the same page!
  • OpenFlights works in realtime. Make any change, and you'll see it right then and there.
  • OpenFlights supports trips. You can join up any flights together into a trip, which you can then display on its own page and even share with friends.
  • OpenFlights is free in spirit. We don't try to lock you in: it's easy to import your data and export copies for safekeeping.
  • OpenFlights is free as in beer. Using the website costs you absolutely nothing, although we reserve the right to nag users for donations if they really get into it — gotta pay the bandwidth bill somehow.
  • OpenFlights is free as in speech. All our source code and (public) data is licensed under the GNU Affero General Public License and can be downloaded from SourceForge, so you can set up your own copy or just use the bits you like.

That's nice, but are you sure this thing will scale?

Yes! The heavy lifting of drawing the maps is handled by your browser, not any central server. The database is built to scale and has been stress-tested by loading over 100,000 flights, but key operations still took under 0.1 seconds each.

Features and bugs

How do I sign up and start punching in my own flights?

At the moment you don't, because the user interface isn't comprehensible/stable enough to let the world go crazy on the flight database quite yet. Stay tuned — it's going to be a lot nicer than what you're used to. If you're really keen on beta-testing, drop us a line at info at openflights dot org and tell us why.

What's the roadmap then?

Roughly like this:
  1. Implement simple flight input (point-to-point).
  2. Implement flight editing.
  3. Add user logins and privacy controls.
  4. Launch public beta.
  5. Implement complex flight input (multi-segment and repeated flights).
  6. Implement importing and exporting.
  7. Add support for filtering according to pretty much anything.
  8. Graphical bells and whistles (eg. color-coding of flights).
  9. Full public launch.

When?

When it's ready.

Can I get a map centred on Australia or the Americas instead of Europe?

The mapping software actually allows wrap-around worlds, so you can center and scroll anywhere, but unfortunately there's a bug in the current version that breaks things. Stay tuned.

Why is the map slow to draw sometimes? Why do map updates slow down my computer?

Map tiles are loaded off the Internet, and the flights, airports etc are drawn on top by your own browser. Switching to a faster computer or just shrinking the window size will speed up things. The NASA geographic tiles are also larger and thus slower than the Metacarta political ones.

Why aren't the flight paths curved?

That would be hard to draw fast, so we cheat and use straight lines. The actual distances, though, are calculated correctly as great circle distances.

Are those flight times estimates accurate?

We currently use a really fancy formula to calculate them: "30 min plus 1 hour per every 500 miles". This seems to be a surprisingly good approximation.

You need to implement cool feature X, and fix terrible bug Y!

Please check SourceForge to see if they're already reported, and add them if not. But remember, this is open source, so the best way to get anything done is to do it yourself!

Technical jibber-jabber

Howdja make that spiffy-keen map?

All credit goes to OpenLayers.

And the rest of it?

JavaScript frontend chock full of AJAX-y goodness, plus PHP scripts on the backend shifting through reams of data pulled from MySQL.

Where did you get all that airline and airport data?

Airport data (c. 5000 airports) was generated by mashing together DAFIF (October 2006 cycle) for location, ICAO data and country data with the Global Airport Database for IATA codes and city names, plus some manual updates for major post-2006 changes.

Airline data (c. 5000 airlines) was extracted directly from Wikipedia's gargantuan List of airlines.

Raw data, cooked data and recipes can be found in the SourceForge SVN repository under data.
